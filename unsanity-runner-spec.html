§ 03애니메이션 명세
ID트리거엔진스펙
MCU-ANI-001 점프 입력 (Space / tap) Three.js 물리 dragon.dy = -13. 중력 0.65/프레임 가속. grounded=false 동안 공중 상태. 지면 도달 시 grounded=true. Dragon Mesh의 position.y로 처리
MCU-ANI-002 달리는 중 (상시) Three.js Dragon Mesh scale.x 1.0 ↔ 1.05 wobble. 8프레임 주기 sin 곡선. mesh.scale.set()으로 처리
MCU-ANI-003 코인 수집 (AABB 충돌) Three.js + DOM 코인 Mesh 위치에 "+10s" DOM 스프라이트 생성 → 40px 위로 fadeup, 600ms 후 제거. #hud-timer-bar opacity 0.5→1 flash 200ms
MCU-ANI-004 충돌 게임 오버 Three.js Dragon mesh.rotation.z 0→2π + mesh.scale 1→0, 600ms ease-in. 완료 후 #gameover-screen 표시. renderer.setAnimationLoop(null) 후 단발 처리
MCU-ANI-005 타임아웃 (time_remaining=0) DOM #hud-timer-bar red pulse 2회 (각 200ms) → width 0으로 300ms shrink → #gameover-screen 표시
MCU-ANI-006 위험 구간 (time_remaining < 30초) DOM CSS #hud-timer-bar 에 CSS class "danger" 추가 → @keyframes pulse: opacity 0.6↔1, 1s infinite. 30초 초과 시 class 제거
⚠️
MCU-ANI-004 / MCU-ANI-005 중복 방지 — 충돌과 타임아웃이 같은 프레임에 발생할 수 있음. phase = gameover 세팅 후 두 번째 트리거 무시. isGameOver 플래그 즉시 set.
💡
MCU-ANI-003 누적 처리 — 여러 코인을 연속 수집 시 "+10s" 텍스트가 겹칠 수 있음. y offset을 각 스프라이트마다 다르게 (index * 16px) 부여하여 분산.
